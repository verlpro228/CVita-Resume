# CVita 在线简历生成平台

CVita 是一个基于 Vue 3 + Vite 构建的纯前端在线简历生成平台，定位为面向普通求职者、在校学生、转岗人群和作品展示用户的 Resume Builder 工具。项目不依赖后端服务，用户可以在浏览器中完成简历创建、模板选择、模块编辑、实时预览、本地保存、PDF 导出和 JSON 数据备份。

本项目不是传统的个人主页或单人简历展示网站，而是一个具有产品化体验的在线工具平台。它的核心目标是降低用户制作简历的门槛，让用户无需注册、无需安装软件，也能快速生成结构清晰、视觉专业、可导出的简历文件。

## 在线演示：https://c-vita-resume-woad.vercel.app/

## 项目背景

在线简历制作是求职场景中的高频需求。传统 Word 简历模板虽然灵活，但存在排版成本高、格式容易错乱、模板复用不方便、跨设备编辑体验差等问题。部分在线简历平台虽然功能完整，但通常需要登录、付费或上传隐私数据。

CVita 采用纯前端方案，将简历数据存储在浏览器本地，兼顾可用性、隐私性和部署成本。项目适合作为毕业设计、课程设计、前端作品集、SaaS 工具型项目演示，也适合作为后续接入真实后端、AI 接口和账户体系的基础版本。

## 产品定位

CVita 是一个在线简历生成平台，属于轻量级 SaaS 工具型产品。

面向用户：

- 应届毕业生：快速创建校招、实习投递简历。
- 社招求职者：根据不同岗位快速调整简历内容和模板。
- 转岗用户：突出项目经历、技能能力和自我评价。
- 前端学习者：作为 Vue + Vite 工具型项目学习案例。
- 教师或评审人员：用于查看前端工程化、交互设计和产品能力。

核心价值：

- 快速制作：打开网页即可编辑，无需复杂配置。
- 实时预览：左侧编辑、右侧简历预览，所见即所得。
- 模板切换：同一份数据可切换多种简历模板。
- 隐私安全：数据默认保存在用户本地浏览器。
- 可导出：支持 PDF 文件导出和 JSON 数据备份。
- 易扩展：统一数据 schema，方便后续接后端或 AI 服务。

## 使用场景

### 首次制作简历

用户第一次进入平台，通过首页了解产品能力，点击“开始制作”进入编辑器。系统提供一份默认示例简历，用户可以直接替换姓名、联系方式、教育背景、项目经历、工作经历、技能等内容，并实时查看右侧预览效果。

### 根据岗位快速调整简历

用户可在已有草稿基础上修改目标岗位、项目描述、技能排序和自我评价。通过模板切换功能，可以针对技术岗、商务岗、学术申请等不同场景选择更合适的简历样式。

### 简历备份与恢复

用户可以将当前简历导出为 JSON 文件，用于本地备份或迁移到其他设备。后续重新打开网站时，可以通过导入 JSON 恢复简历内容。

### 投递前导出 PDF

用户完成内容编辑和模板选择后，可将右侧 A4 预览区域导出为 PDF 文件，用于邮件投递、招聘平台上传或线下打印。

### 前端作品展示

项目具备完整的产品首页、模板中心、编辑器、实时预览和导入导出流程，适合作为前端工程化和交互设计能力展示项目。

## 非功能需求

### 性能要求

- 首页和模板页应快速加载。
- 编辑器表单输入应保持流畅。
- PDF 导出时需要提供状态反馈，避免用户误以为页面卡死。
- 第三方库按需分包，减少单个构建产物过大。

### 兼容性要求

- 支持现代浏览器，例如 Chrome、Edge、Safari、Firefox。
- 支持 PC、平板和移动端浏览。
- 编辑器在移动端可降级为编辑/预览切换形式，保证基本可用。

### 可维护性要求

- 数据模型集中管理。
- 模板数据与页面逻辑分离。
- 预览组件与编辑组件解耦。
- 样式保持统一，避免大量一次性临时样式。
- 组件拆分清晰，便于后续扩展。

### 安全与隐私要求

- 当前版本不上传简历数据。
- 用户数据保存在本地浏览器。
- JSON 导入仅解析用户主动选择的本地文件。
- 外部链接通过新窗口打开，并使用安全的跳转策略。

## 技术栈说明

### 前端框架

- Vue 3：用于构建组件化用户界面。
- Composition API：用于组织复杂页面状态和逻辑。
- `<script setup>`：提升 Vue 单文件组件编写效率。

### 构建工具

- Vite：负责开发服务器、模块热更新和生产构建。
- `@vitejs/plugin-vue`：提供 Vue 单文件组件编译能力。

### 路由

- Vue Router：负责页面路由管理。
- 当前使用 history 模式，使线上路径更清晰，例如 `/editor`、`/templates`。
- 通过 `vercel.json` 配置 SPA rewrite，解决刷新 404 问题。

### UI 与样式

- Element Plus：提供基础 UI 组件和图标能力。
- Sass：用于增强样式组织能力。
- Tailwind CSS：项目保留 Tailwind 配置，可用于工具类样式扩展。
- 自定义 CSS：用于实现产品级浅色玻璃质感、响应式布局和简历模板样式。

### 动画

- GSAP：用于首页入场动画、卡片微交互、编辑器切换动效和 UI 过渡效果。

### 导出能力

- html2canvas：将简历预览区域转换为画布图像。
- jsPDF：将画布内容写入 PDF，实现浏览器端 PDF 导出。

### 数据持久化

- localStorage：保存当前简历草稿、主题设置和布局配置。
- JSON 文件：用于简历数据备份、迁移和恢复。

### 自动导入

- unplugin-auto-import：自动导入常用 API。
- unplugin-vue-components：自动注册组件，减少重复 import。

## 简历数据模型

项目使用统一的 resume schema 管理简历数据，便于本地保存、导入导出和模板渲染。

核心结构包括：

```js
{
  schemaVersion: 1,
  id: 'resume-id',
  title: '简历名称',
  templateId: 'modern',
  theme: {
    accent: '#111827',
    density: 'comfortable',
    fontFamily: 'system',
    lineHeight: 1.5,
    baseFontSize: 13,
    sectionTitleSize: 16,
    itemTitleSize: 15,
    pagePaddingX: 62,
    pagePaddingY: 56,
    sectionGap: 28,
    itemGap: 18,
    paragraphGap: 8,
    headerLayout: 'balanced'
  },
  basics: {
    name: '',
    position: '',
    phone: '',
    email: '',
    city: '',
    address: '',
    website: '',
    github: '',
    avatar: '',
    summary: ''
  },
  sections: [
    {
      id: 'section-id',
      type: 'projects',
      title: '项目经历',
      visible: true,
      items: []
    }
  ]
}
```

设计优势：

- 所有模块统一存放在 `sections` 中。
- 模块支持排序、显示隐藏和动态扩展。
- 模板只依赖 schema 渲染，不依赖编辑器内部状态。
- JSON 导入导出结构清晰，方便后续接后端。

## 模板系统

当前模板系统基于 `templateId` 和 `theme` 配置共同驱动。

模板方向包括：

- 现代简约：适合通用岗位和校招场景。
- 经典商务：适合传统行业、管理岗位和正式场景。
- 技术专业：强调项目经历、技术栈和技能展示。
- 创意作品：适合作品集、设计、运营等岗位。
- 学术研究：适合升学、科研和学术申请。
- 极简单页：适合信息密度较高的一页式简历。

模板切换要求：

- 同一份数据切换模板后，预览即时变化。
- 不同模板在头部布局、模块标题、强调色、分隔样式和内容密度上体现差异。
- 模板不应破坏 resume schema 的通用性。

## 当前版本边界

当前版本为纯前端演示版，因此存在以下边界：

- 不包含用户登录和账号体系。
- 不支持跨设备云同步。
- AI 功能为前端 mock，并未接入真实大模型接口。
- 简历数据仅保存在当前浏览器 localStorage 中。
- PDF 导出依赖浏览器环境，复杂内容分页仍可继续增强。

这些边界不影响项目作为在线简历生成工具的核心演示，但后续如果要上线为真实产品，可以继续扩展。
